# Privacy Policy

## Overview

KanTrack is a local-first personal workflow tool.

By default, all data is stored locally on the user's device. No network connection is required. No accounts exist. No data is ever transmitted.

---

## Current Deployment

KanTrack is deployed as a fully static site (Cloudflare Pages). There is no backend, no server, and no database outside the user's own browser. The app itself is served as static files — it cannot send or receive user data.

---

## Local Mode (Default and Only Mode)

In local mode:

- Data is stored entirely on the user's device (IndexedDB + localStorage)
- No account is required
- No data is transmitted externally
- No tracking or analytics occur
- No cookies are used for tracking

---

## Optional Cloud Mode (Future — Not Yet Available)

A future version may offer an optional, explicit cloud backup feature. If and when implemented:

- It will be strictly opt-in
- The user will initiate it manually
- No behavioral analytics will be collected
- Only the user's own KanTrack data will be involved

Until that feature exists, none of this applies.

---

## Data Collection

KanTrack does not collect:

- Personal information
- Usage analytics
- Tracking data
- Behavioral metrics
- IP addresses or device identifiers

---

## Browser Storage

Local browser storage (IndexedDB, localStorage) is used only to persist the user's tasks, notes, and preferences on their own device. No tracking is involved.

---

## Data Ownership

Users retain full control of their data and may export or delete it at any time from within the app.

---

## Source Transparency

KanTrack is source-available. Users can read every line of code to verify how their data is handled.

---

## Philosophy

Offline by default. Online by choice.
