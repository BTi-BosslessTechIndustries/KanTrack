# scripts/kantrack-modules/

Every feature module of the KanTrack application. Each file has one responsibility.

---

## TypeScript core (typed infrastructure)

| File            | What it does                                                                                                                                               |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `types.ts`      | Shared domain types: `Task`, `Column`, `Priority`, `Tag`, `NotebookItem` (incl. `expanded?: boolean`), `Clock`, `UndoEntry`                                |
| `store.ts`      | Minimal flux-like store — `dispatch()`, `subscribe()`, `getState()`, typed reducer, `TASK_*` action types                                                  |
| `router.ts`     | Event delegation router — `registerAction()` maps `data-action` names to handlers; `initRouter()` attaches one document-level click listener               |
| `repository.ts` | **All** persistence logic: `getAllTasks()`, `saveTasks()`, `getAllNotebookItems()`, etc. Writes localStorage synchronously and IDB async (fire-and-forget) |

## Key data flows

### Creating a task

`addNote()` (tasks.js) → `state.pushToNotesData()` → `saveNotesToLocalStorage()` → `repository.saveTasks()` → localStorage (sync) + IDB (async)

### Saving modal changes

Both paths share `_persistModalChanges()` which handles title, priority, tags, timer, and note content:

- **Save & Close:** `saveAndCloseModal()` → `_persistModalChanges()` → `saveNotesToLocalStorage()` → `updateNoteCardDisplay()` → `closeTaskModal()`
- **Save (keep open):** `saveModal()` → `_persistModalChanges()` → `saveNotesToLocalStorage()` → `updateNoteCardDisplay()` → resets tracking state + clears editor + re-renders history in place

### Loading on startup

`loadNotesFromLocalStorage()` (storage.js) → `repository.getAllTasks()` → localStorage (primary, always written synchronously) or IDB (fallback) → `state.setNotesData()` → render all cards
