# Contributing to KanTrack

Thank you for your interest in improving KanTrack.

KanTrack is a privacy-first, local-first personal workflow tool designed to work entirely offline by default, with optional cloud features available only if explicitly enabled by the user.

---

## Philosophy First

KanTrack is built around:

- Individual-first design
- Local-first data storage
- Privacy by default
- Offline capability
- Clarity over feature expansion

Changes that compromise these principles will not be accepted.

---

## What Contributions Are Welcome

You may contribute:

- Bug fixes
- Performance improvements
- Security enhancements
- Accessibility improvements
- Documentation improvements
- UI clarity improvements

---

## What Will Likely Not Be Accepted

The following types of changes are generally out of scope:

- Features requiring mandatory accounts
- Tracking, analytics, or telemetry
- Team-management features
- Surveillance or metrics systems
- Changes that increase complexity without improving clarity

---

## Before Submitting a Contribution

Please:

1. Open an issue describing the problem or improvement
2. Explain your proposed approach
3. Wait for feedback before large changes

---

## Code Guidelines

- Keep changes focused and minimal
- Preserve readability and simplicity
- Avoid unnecessary dependencies
- Respect the local-first architecture

---

## Security & Privacy

Contributions must not introduce:

- External data transmission without explicit user action
- Hidden storage
- Background network requests
- Any form of user tracking

---

## License Agreement

By contributing, you agree that your contributions will be part of the KanTrack project under the same license.

---

Thank you for helping improve KanTrack.
